# 把学习痕迹交给 Git 的那个春天

那时候我对 Git 的理解还很朴素：保存代码，必要时回退，偶尔把仓库推到 GitHub 上，显得自己像一个正在认真学习的人。

后来才发现，Git 最有用的地方不是“看起来专业”，而是它会诚实记录你怎么变菜、怎么修正、怎么从一个命名随缘的人变成至少愿意写清楚提交信息的人。它不会鼓励你，但也不会嘲笑你。提交历史静静躺在那里，像一排不太整齐的脚印。

大二的春天，我开始把课堂练习、后端 demo、读过的项目结构放进仓库。很多代码现在回头看会有点想捂脸：目录乱、注释少、变量名像临时起意。但它们是某种起点。那时还没有真正进入 RAG、Agent 或大模型应用，只是在慢慢补工程地基。

我也开始理解一个事实：学习不是等“完全会了”再记录，而是在还不完全会的时候就记录。因为真正有价值的，往往不是最后那个能跑的版本，而是中间那些“为什么跑不起来”的痕迹。

那一年我给自己留下的第一个 checkpoint，大概就是：别急着把自己包装成已经抵达的人，先成为一个愿意留下路线的人。
