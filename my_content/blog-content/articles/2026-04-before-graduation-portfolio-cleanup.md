# 毕业前整理作品集：把项目线从硬盘里救出来

毕业前整理作品集，感觉像清理一间住了四年的房间。文件夹很多，仓库很多，截图很多，命名风格也很多。最可怕的是，有些项目当时做得很认真，但过几个月再打开，自己都要先读 README 才想起来入口在哪里。

我开始把项目分成几类：公开仓库、实习项目、学习痕迹、兴趣实验。公开仓库可以直接链接，实习项目要注意边界，只讲适合公开讨论的技术内容；学习痕迹不用强行包装成成熟产品；兴趣实验可以保留，但不写成正式上线项目。

这件事比想象中难，因为它不是简单罗列经历，而是在回答“我到底是怎么走到这里的”。SecondClass 和 Spring Boot 是工程起点，RAG 项目是系统意识，Whisper 插件是语音支线，TCM-Health-Bot 是微调到应用，KnowBot 和 KARE-Med 则把 Agent、RAG 和安全治理收束起来。

整理过程中也会发现很多遗憾：有些 README 不够清楚，有些截图缺失，有些功能当时没留下演示。但这些遗憾本身也提醒我，项目完成后要及时归档。

作品集不是把自己包装得完美，而是把真实路线讲清楚。
