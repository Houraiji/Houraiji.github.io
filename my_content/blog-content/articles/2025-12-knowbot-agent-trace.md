# KnowBot 的 Agent Trace：让智能体别只会神秘地回答

做 KnowBot 时，我越来越不喜欢那种“模型直接给出一段答案”的黑箱体验。企业知识问答不是许愿池，用户真正想知道的不只是答案本身，还有答案从哪里来、经过了哪些步骤、是否能被追溯。

Agent Trace 就是在这个背景下变得重要。一次查询可能经历意图识别、查询改写、知识库检索、工具调用、结果汇总和答案生成。如果这些步骤完全不可见，系统出错时就很难定位：是问题理解错了，还是检索没召回，还是工具路由走偏了？

引用溯源也是企业知识场景的底线之一。回答里最好能给出来源、匹配片段或证据入口，让用户知道系统不是凭空组织语言。对于内部知识库来说，这种可信度比“回答看起来很顺”更重要。

LangGraph 一类编排思路让我开始把 Agent 看成状态流，而不是一个更会聊天的 prompt。节点、边、条件路由、工具执行结果，都可以被记录、展示和调试。

KnowBot 给我的结论是：智能体不是越神秘越高级。相反，越能解释自己怎么工作的系统，越接近可交付。
